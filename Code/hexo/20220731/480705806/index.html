【hexo建站】 六: 优化文章路径,各大搜索引擎接入

发表于 2022-07-31 分类于 Code ， hexo建站 Valine：
 简介：SEO对中文路径识别没有英文好所以做以下修改 ，各大搜索引擎接入；

1： 美化文章路径

1 输入： npm install hexo-abbrlink --save
2 站点配置文件中搜 permalink  修改格式 category/:year:month:day/:abbrlink/
3 在末端加入

4 分类调整格式： category_map: hexo: hexo 
5 hexo clean g s  查看效果
6 文章md文件下会添加abbrlink : xxxx
7 blog - public 可以看到 hexo建站 的分类文件夹被重命名为hexo了
8 http://localhost:4000/categories/hexo/    文章地址也没有中文了

2： SEO接入

1 主题配置文件中搜SEO Settings 
2 可以下拉 关键字 google bing(必应) yandex(俄罗斯搜索引擎) baidu

谷歌 verification ：

👉 谷歌

1 谷歌登入 输入网站地址，如 https://linsheng.tk
2 网域 或者 网址前缀(IO的用)
3 网站验证 ， 复制一串代码 到google_site_verification:     
4 提交修改到github ，验证成功

👉 必应

1 必应简单点，直接用谷歌的数据。
2 导入谷歌就可以了

👉 Yandex

1 Yandex谷歌账号登入 输入网站地址，如 https://linsheng.tk
2 步骤大同小异
3 验证，下载html文件到next/source下
4 提交修改
5 验证成功

👉 百度

1 百度账号登入 输入网站地址，如 https://linsheng.tk
2 步骤大同小异
3 验证，下载html文件到next/source下
4 提交修改
5 验证成功

👉 sitemap站点地图

官方仓库sitemap ： 

1 npm install hexo-generator-sitemap --save   blog下
2 主题配置中搜 menu  开启sitemap: /sitemap.xml || sitemap
3 hexo clean  hexo g  就可以在blog\public  看到sitemap
4 提交到SEO优化默认 网址/sitemap.xml

👉 robots 蜘蛛协议

robots.txt（统一小写）是一种存放于网站根目录下的文本文件
它通常告诉网络搜索引擎的漫游器（又称网络蜘蛛）
此网站中的哪些内容是不应被搜索引擎蜘蛛获取的，哪些是可以被（蜘蛛）获取的。
